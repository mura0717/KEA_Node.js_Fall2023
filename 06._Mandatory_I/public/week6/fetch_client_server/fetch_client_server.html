Fetching in Client-side vs server-side

Client-side data fetching does not need to pre-render your data when the content of your pages needs to update frequently.

On the other hand, Server-side rendering (SSR) allows to render pages on the server, rather than in the browser.

Client-Side Fetch:


                        app.get('/proxyserver', (req, res) => {
                            fetch('https://api.example.com/data')
                              .then((response) => response.text())
                              .then((result) => res.send(result))
                              .catch((error) => {
                                console.error(error);
                                res.status(500).json({ error: 'Internal Server Error' });
                            });
                        });

Server-Side Fetch:


                        fetch('/api/data')
                            .then((response) => {
                                if (!response.ok) {
                                throw new Error('Network response was not ok');
                                }
                                return response.json();
                            })
                            .then((data) => {
                                console.log(data); // Process the fetched data
                            })
                            .catch((error) => {
                                console.error('Fetch error:', error);
                            });
