Redirection: Client-Side vs. Server-Side

In JavaScript, there are several ways to perform redirections.
These essentially involve changing the current URL of a web page to navigate to a different location.

Client-Side Redirections:

The "window.location" object stores the location or URL of the page,
and we can redirect into another page using this window.location. 


        //to set or return the complete URL of the current page.
        window.location.href = "https://example.com";

        //to replace the current document with the specified one.
        window.location.replace("https://example.com");

        //to load a new document.
        window.location.assign("https://example.com");

Server-Side Redirections:

A server-side redirect is a forwarding method in which the server sends a 3xx HTTP status code when a URL is requested.
Server-side redirections involve the server instructing the client to navigate to a different URL: 


        // Redirect to a new URL with a 302 status code
        res.redirect(302, 'https://example.com');
        
        //Redirect by using an HTML  with specifying a delay.
